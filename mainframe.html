Consider the following relational table for maintaining contact information:
給定如下關聯式表格以儲存並維護聯絡人資訊 

create table contacts (
  lname  varchar(30),
  fname  varchar(30),
  email  varchar(30),
  homePhone varchar(20),
  cellPhone varchar(20),
  officePhone varchar(20),
  address varchar(100),
  comment varchar(100),
  primary key (lname,fname)
);
Design and implement a set of HTML/PhP pages that allow the user to
設計並以 HTML/PHP 實作下列網頁功能:
  1. ADD a contact 新增連絡人
  2. DELETE one or more contacts: You need to show all the contacts in a tabular form with check boxes next to each contact; The user will choose contacts to delete by checking the checkboxes and Submitting the form. 刪除一至多筆連絡人: 顯示所有連絡人並在每個項目旁加一個複選框. 使用者可勾選要刪除的連絡人並按下送出按鈕.
  3. UPDATE one or more contacts: You need to show all contacts in a tabular form with check boxes next to each contact; The non-key columns should be displayed as text boxes so that the user may change values; The user chooses contacts to be updated and provides new values in the text boxes and submits. 更新一至多筆連絡人: 顯示所有連絡人並在每個項目旁加一個複選框. 使用者可勾選要更新的連絡人並按下送出按鈕. 所有非鍵值欄位皆以輸入文字框顯示, 以便使用者更改其值.
  4. SEARCH by keyword: Search only fname or lname. 搜尋鍵值: 輸入 fname 或 lname 並顯示符合條件的連絡人.
  5. LIST all contacts (one page with just fname and lname - hyperlinked); Hyperlinks lead to a detailed page. 列出所有連絡人的姓名; 點按之後以超連結顯示細節.
